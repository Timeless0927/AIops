AIOps Console V1 视觉方向方案

同一组 Kubernetes 告警、Incident 详情、飞书原生审批和成本语义，用 4 套界面方向横向比较。所有方案都按企业级作战台处理：高信息密度、少装饰、状态清晰、便于值班 SRE 长时间扫描。

默认推荐：方案 A「Quiet Ops」 最适合 AIOps Console V1：浅色运维控制台学习成本低，状态色不依赖深色氛围，表格/详情/审批信息密度均衡，后续 AIO-88 到 AIO-92 页面可以低成本复用导航、列表、详情、侧栏和审计组件。

A. Quiet Ops

偏浅色的标准企业控制台。信息密度高、实现成本最低，适合 V1 默认落地。

B. Control Room

深色作战室方向，适合大屏和夜间值守。异常冲击力强，但表单/审批长文本需要更谨慎的可读性校验。

C. Audit Ledger

强调审计、审批链和证据留痕。适合安全合规汇报，但日常操作会比 A 更显厚重。

D. Signal Board

强调信号分层和拓扑归因。适合后续扩展拓扑/依赖视图，实现复杂度略高。

A. Quiet Ops / 浅色运维控制台

冷静、清晰、低学习成本。用浅底、细分割线和克制的青绿色强调当前作战对象，告警、审批、成本和审计信息都保持可扫描。

推荐作为 V1 默认方向。信息密度、可扩展性和实现成本最均衡，后续列表页、详情页、审批中心和成本面板都能沿用同一套组件。

B. Control Room / 深色作战室

暗底和琥珀强调让异常更像作战信号。适合夜间值守、大屏和高优先级 incident war-room，但审批表单长文本需要额外做对比度与疲劳测试。

状态表达最强，氛围最贴近“作战台”。实现成本中等；若 V1 主要在普通办公显示器和飞书场景使用，不建议作为唯一默认主题。

C. Audit Ledger / 审计台账

用台账式分割、暖灰底和棕色动作强调“谁批准了什么、何时批准、系统按什么边界执行”。适合强调安全审查和审批闭环的客户演示。

合规可信度强，审计信息很突出。日常值班操作会略显正式厚重；如果后续页面以快速处置为主，建议只吸收其审批和审计模块语言。

D. Signal Board / 信号分层看板

用深色左轨承载集群边界，主画布保持浅色；状态色从“告警噪声”转成“信号层级”。适合后续把 topology、evidence 和 dependency impact 做成第一等视图。

扩展潜力好，视觉记忆点明确。实现上需要更严格的组件约束，避免左轨深色和主区浅色割裂；不建议在 V1 时间紧时一次铺满所有页面。
